# Markdown格式化指南

## 📝 文章格式优化标准

本文档说明如何格式化文章内容，使其更易读、更专业、更符合SEO最佳实践。

## ✅ 已优化的格式元素

### 1. 标题层次

使用清晰的标题层次结构：

```markdown
# H1 - 文章标题（frontmatter中已有，正文中可省略）

## H2 - 主要章节
### H3 - 子章节
#### H4 - 细分内容（少用）
```

**最佳实践：**
- H1 只在 frontmatter 的 title 中使用
- H2 用于主要章节（3-6个）
- H3 用于子章节和列表项
- 保持标题层次清晰，不要跳级

### 2. 段落和分隔

**段落间距：**
- 段落之间空一行
- 列表前后各空一行
- 代码块前后各空一行

**视觉分隔：**
```markdown
---  # 用于主要章节之间的分隔
```

### 3. 列表格式

**无序列表：**
```markdown
- **加粗关键词** - 描述内容
- **另一个要点** - 更多信息
```

**有序列表：**
```markdown
1. **步骤一** - 详细说明
2. **步骤二** - 详细说明
3. **步骤三** - 详细说明
```

**嵌套列表：**
```markdown
- 主要点
  - 子点一
  - 子点二
```

### 4. 强调和加粗

**使用场景：**
- **加粗（**text**）**：关键词、重要概念、步骤标题
- *斜体（*text*）*：强调、引用、术语
- `代码（`code`）`：技术术语、命令、文件名

**示例：**
```markdown
The **"Something went wrong"** message is a generic error notification.
```

### 5. 引用块

**使用场景：**
- 重要提示
- 注意事项
- 关键信息

**格式：**
```markdown
> **Note:** This is an important note.
> **Important:** Pay attention to this.
> **Tip:** Here's a helpful tip.
```

### 6. 代码块

**行内代码：**
```markdown
Press `F5` to refresh the page.
```

**代码块：**
````markdown
```javascript
// 代码示例
const example = "code";
```
````

### 7. 链接

**内部链接：**
```markdown
Check out our [troubleshooting guides](/troubleshooting-guides).
```

**外部链接：**
```markdown
Visit [ChatGPT Status Page](https://status.openai.com).
```

### 8. 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
```

## 🎨 格式化最佳实践

### 1. 开头段落

- 第一段应该直接回答问题
- 包含关键词和核心信息
- 简洁明了，不超过3-4句

### 2. 章节结构

**推荐结构：**
1. **快速答案**（可选）
2. **问题解释**
3. **原因分析**
4. **解决方法**
5. **常见误解**
6. **何时需要担心**
7. **总结**

### 3. 列表优化

**使用加粗关键词：**
```markdown
- **关键词** - 详细说明
```

**使用emoji增强可读性（可选）：**
```markdown
- ✅ 正确做法
- ❌ 错误做法
- ⚠️ 注意事项
```

### 4. 视觉元素

**分隔线：**
```markdown
---  # 用于主要章节之间
```

**空行：**
- 段落之间：1个空行
- 章节之间：2个空行
- 列表前后：各1个空行

### 5. 强调技巧

**重要信息加粗：**
```markdown
**Important:** This is crucial information.
```

**步骤编号：**
```markdown
1. **Step 1:** Description
2. **Step 2:** Description
```
